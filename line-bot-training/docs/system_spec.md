# システム仕様書：LINEボット ＆ アルバイト自動研修評価システム

本ドキュメントは、AI技術を活用した店舗自動応答LINEボット、およびアルバイト向けの自動研修・スキル評価システムの機能・技術・運用仕様を定義したものです。

---

## 1. システム概要

### 1.1 目的
- **店舗業務の効率化**: 店舗への基本的な問い合わせ（営業時間、メニュー、アクセス等）をAI店員が24時間自動応答します。
- **研修・評価コストの削減**: アルバイトがLINE上でマニュアルの理解度テストを受け、自動採点と評価通知を行うことで、管理者の手動評価の手間を削減します。
- **納品・導入の高速化**: 重い外部ライブラリを排除した軽量なプログラム構成により、短納期での環境構築・横展開を可能にします。

### 1.2 主なユーザー
- **一般顧客**: LINE上で店舗への問い合わせを行う。
- **アルバイト従業員**: LINE上でマニュアル研修テストを受験する。
- **店舗管理者（マネージャー）**: テスト結果の自動通知を受け取り、時給評価や実案件アサインを判定する。

## 3. 機能仕様

### 3.1 店舗AI自動応答機能
- **起動条件**: ユーザーが通常のメッセージ（例:「営業時間は？」）を送った場合。
- **動作仕様**:
  - 外部ファイル `cafe_info.md` に記載された店舗基本情報を参照し、Gemini APIを用いて自然な日本語で回答を生成します。
  - 店舗情報に記載のない質問に対しては、適当な嘘をつかず「分かりかねます」と回答します。

### 3.2 アルバイト自動スキルテスト機能
- **起動条件**: ユーザーがトーク画面で **「テスト開始」** と送信した場合。
- **対話フロー**:
  1. **氏名登録**: フルネームの送信を要求します。
  2. **テスト出題**: `skills_test.json` から選択式問題（A, B, C, D の4択）を順次1問ずつ出題します。
  3. **回答受信と採点**: アルバイトの回答（英字）をリアルタイムで検証・採点します。
  4. **合否判定**: 30点満点中 **20点以上（2問以上正解）で合格** と判定します。
  5. **結果表示**: 得点および判定結果（合格🎉 / 不合格😢）をアルバイトの画面に返信します。

### 3.3 管理者通知 ＆ 結果蓄積機能
- **結果の保存**: 受験日時、受験者氏名、得点、合否判定を `test_results.json`（本番では Google スプレッドシート）へ自動保存します。
- **自動通知**: 受験完了と同時に、マネージャー宛てに結果報告メッセージ（時給アップ対象か否か）を自動送信します。

### 3.4 Dialogflow CX 互換 Webhook 機能
- **概要**: Google Cloud の Dialogflow CX（ビジュアル会話設計ツール）から直接呼び出せる Webhook エンドポイント（`POST /dialogflow/webhook`）を提供します。

## 4. API ＆ データ仕様

### 4.1 エンドポイント一覧

| パス | メソッド | 説明 |
| :--- | :--- | :--- |
| `/webhook` | `POST` | LINE Messaging API からの通知を受信するメインエンドポイント。 |
| `/dialogflow/webhook` | `POST` | Dialogflow CX からのセッションパラメータを受け取り、採点結果を返すエンドポイント。 |
| `/debug/chat` | `POST` | ローカル環境でLINE接続なしに動作検証を行うデバッグ用エンドポイント。 |

### 4.2 環境変数 (Secrets) 仕様

| 変数名 | 必須 | 説明 |
| :--- | :--- | :--- |
| `LINE_CHANNEL_SECRET` | ○ | LINE Developers のチャネルシークレット |
| `LINE_CHANNEL_ACCESS_TOKEN` | ○ | LINE Developers のアクセストークン |
| `GEMINI_API_KEY` | ○ | Google Gemini API キー (未設定時はモック応答モード) |

---

## 5. 動作環境 ＆ 依存関係

- **実行言語**: Python 3.9 以上
- **主要ライブラリ**:
  - `flask`: Webサーバー基盤
  - `requests`: HTTP通信（Gemini API 呼び出し、LINE API 返信用）
- **環境構築の特徴**: C/C++やRustのコンパイルを必要とするヘビーなSDKを一切使用しないため、どんな環境でも失敗なく数秒でセットアップ可能です。
