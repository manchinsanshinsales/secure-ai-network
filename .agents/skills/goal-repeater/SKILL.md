# Goal Repeater スキル

このスキルは、ユーザーから `/goal` コマンドが指定された際、あるいは長期実行が必要なタスクにおいて、エージェントが自律的に検証、自己修正、および品質向上ループを繰り返すことをガイドするための指示セットです。

## 出力言語の制約

- 本スキルに基づく実行中、思考プロセス、ログ、生成するドキュメント、およびユーザーへの応答を含め、すべての出力は**日本語**で行ってください。

## 指針と実行フロー


1. **ゴールの細分化とタスク管理 (`task.md`)**:
   - ユーザーの要望やゴールを達成するために必要なタスクを、具体的な検証可能な項目に分解して `task.md` にリスト化します。
   - 進捗状況を随時 `[ ]`, `[/]`, `[x]` で更新し、漏れがないか確認します。

2. **徹底した検証コード・スクリプトの作成**:
   - 変更や実装を行う際、単に動作を推測するのではなく、検証用のスクリプト（テストコードや確認用のコマンド）を `scratch/` ディレクトリ配下等に作成します。
   - 作成したスクリプトを実行し、実際の出力を分析します。

3. **自己修正ループ (Self-Correction Loop)**:
   - テストや検証が失敗した場合、あるいは想定外の挙動が見つかった場合は、ログやコードを詳細に分析して修正します。
   - 修正後、再度検証を実行し、エラーが解消されるまでこのループを繰り返します。

4. **エッジケースの追求**:
   - 正常系が動作しただけで満足せず、境界値、例外処理、空データ、パフォーマンスの問題など、潜在的なエッジケースを洗い出してテストを行います。

5. **最終品質チェックと完了判定**:
   - すべてのタスクが完了し、作成した検証スクリプトがすべて合格することを確認します。
   - 自信を持って「ゴールを完全に達成した」と証明できる状態になったら、応答の末尾に `<!-- GOAL_COMPLETE -->` を付与して終了します。
