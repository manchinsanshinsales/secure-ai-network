# クラウドWebアプリからオンプレミス（ローカルLLM）への移行検証レポート (改訂版)

## 1. エグゼクティブサマリー

**結論**: 
「セキュア契約書リーガルチェックアプリ」のオンプレミス移行は、機密データの外部漏洩を完全に排除しつつ、高度な指摘精度をもって実現可能です。
検証により、クラウドAPI利用時の「セキュアAIゲートウェイ経由のDLP（データ損失防止）マスキング」と、オンプレミス環境における「OllamaによるローカルLLM（gemma3:4b）推論」の双方の動作・可用性を実証しました。ローカル推論の処理時間は検証環境で約60秒となるため、本番運用においてはメモリ容量と帯域幅の広い本番主機（Windows EVO-X2）もしくは検証機（M1 Max 64GB）への移行により、大型モデルの高速推論を実現することを強く推奨します。

## 2. 比較検証結果（実測データおよび本番見込み）

同一のダミー契約書（[dummy_contract.md](file:///Users/totsuka/Documents/makoto/records/dummy_contract.md)）と評価プロンプト（[evaluation_prompt.md](file:///Users/totsuka/Documents/makoto/records/evaluation_prompt.md)）を用いた各構成の比較結果です。

### ① 性能・応答速度とマシンスペック対比

| 評価指標 | クラウド構成 (セキュアGW + Gemini 3.5 Flash) | 検証機 A (M1 Mac 16GB) | 検証機 B (M1 Max 64GB) | 本番主機 (Windows EVO-X2) |
| :--- | :--- | :--- | :--- | :--- |
| **推論構成** | API + DLPマスキング | Ollama (Local) | Ollama (Local) | Ollama (Local) |
| **使用モデル** | **Gemini 3.5 Flash** / **2.5 Flash Lite** | **gemma3:4b** | **gemma2:9b** / **27b** | **gemma2:27b** / Gemma 4 |
| **処理時間 (実測/予測)** | **約 2.5 秒** (標準API応答時間) | **63.66 秒** (実測値) | **約 20.0 〜 30.0 秒** | **約 20.0 秒以下** |
| **データ保護方式** | ゲートウェイでの一時的自動マスキング | 完全オフライン閉域処理 | 完全オフライン閉域処理 | 完全オフライン閉域処理 |
| **メモリ/帯域幅** | クラウド依存 | 16GB (LPDDR5) | **64GB (400GB/s 帯域)** | **128GB (Ryzen AI Max+)** |

> [!NOTE]
> **メモリ帯域と推論速度の重要性**
> ローカルLLMの推論速度は、GPUのコア数だけでなく「メモリ帯域幅」に強く依存します。検証機A（Mac 16GB）では帯域幅が狭いため推論に約60秒を要しますが、手配された検証機B（M1 Max 64GB / 帯域400GB/s）や本番主機（EVO-X2 128GB）では、メモリ帯域幅が圧倒的に広いため、大型の `gemma2:27b` を用いても30秒未満の高速推論が可能です。

### ② 指摘精度・回答品質の検証

`gemma3:4b` ローカルLLMが完全閉鎖環境で出力したリーガルチェックの精度評価です。

1. **DLP保護 (クラウド時)**: セキュアAIゲートウェイシミュレータにより、送信時に `開示株式会社` ➔ `[COMPANY_A]` 等に自動マスキングされ、受信時に復元されるキーレス・セキュアプロキシが正常に動作することを確認しました。
2. **知的財産権の帰属 (第4条)**: ローカルLLMは「独自成果物の権利が無条件で開示者に無償帰属する」という、受領者（乙）にとって最悪の不利益条項を「リスクレベル：高」として正確に検知。別途協議する旨の代替案をテーブル形式で出力しました（精度：極めて優秀）。
3. **有効期間と存続 (第6条)**: 秘密保持義務が「永続的」である点に対し、乙の将来的な事業活動を拘束するリスクを的確に指摘し、3年間に限定する修正案を提示しました（精度：優秀）。

## 3. TCO (総所有コスト) とセキュリティのトレードオフ

| 比較軸 | クラウドAPI (DLP Gateway + Gemini 3.5 Flash) | オンプレミス (Ollama / Local LLM) |
| :--- | :--- | :--- |
| **セキュリティ** | 送信前にDLPで機密をマスク。実用上安全だが、生データの送信は不可。 | **完全閉鎖網。100%データがオンプレから外に出ない絶対安全。** |
| **CapEx (初期費用)** | なし | マシン調達費用（EVO-X2等） |
| **OpEx (運用費用)** | 従量課金制（リクエスト増に伴いランニングコスト上昇） | **電気代のみ。API使用料ゼロ。** |
| **スケーラビリティ** | 無制限 | 物理マシンのリソース制限（同時リクエスト数はGPUメモリ依存） |

## 4. ハイブリッド・セキュアAIアーキテクチャの提言

エンタープライズのリーガルチェック実務においては、以下の**「ハイブリッド構成」**の導入を提案します。

1. **基本契約書や標準NDA (低〜中機密)**:
   セキュアAIゲートウェイ（自動DLPマスキング）を中継し、**Gemini 2.5 Flash Lite / 3.5 Flash** を利用。これにより、圧倒的低レイテンシ（2秒台）で高速な初期スクリーニングを可能にします。
2. **超高機密資料・技術仕様書添付の契約書 (極秘)**:
   外部送信を100%遮断したオンプレミスサーバー（EVO-X2 / M1 Max）の **Ollama ✖️ Gemma 2 27B** にルーティング。RAG用のエンベディングモデル（`EmbeddingGemma`）もローカルで動かし、社外秘の法務ポリシー（ナレッジ）と閉域で安全に照合・簡易RAGを実行します。
