# クラウドWebアプリからオンプレミス（ローカルLLM）への移行検証レポート

## 1. エグゼクティブサマリー

**結論**: 
「セキュア契約書リーガルチェックアプリ」のオンプレミス移行は、データの外部送信を完全に排除するセキュリティ要件を満たしつつ、実用的な指摘精度をもって実現可能です。検証の結果、M1 Mac (16GB) であっても `gemma3:4b` ローカルLLMを用いることで、NDAの重要論点を漏れなく抽出できることを確認しました。ただし、ローカル推論の処理時間はクラウド比で約20倍（約56秒）となるため、本番環境では強力なNPU/GPUを搭載した主機（Windows EVO-X2）への移行による処理速度の最適化を強く推奨します。

### ② 指摘精度・回答品質比較

`gemma3:4b` がローカル環境で出力した契約書リーガルチェック結果（実数値）の評価は以下の通りです。

1. **第1条（秘密情報）の指摘**: 「書面明示に限定されている点」「30日以内の通知が必要な点」について、乙（受領者）にとって不利であると正しく抽出し、通知期限を90日に延ばす代替案を提示（精度：優秀）。
2. **第4条（知的財産権）の指摘**: 最も高リスクな「独自開発成果物の開示者への無償帰属」を「リスクレベル：高」として正確に検知し、「別途協議の上合意する」という実務的な修正案を出力（精度：極めて優秀）。
3. **第5条（損害賠償）の指摘**: 賠償上限が100万円または過去6ヶ月の取引総額の低い方という受領者側の不利に対し、期間を12ヶ月、上限を300万円に引き上げる現実的な対案を出力（精度：優秀）。
4. **第6条（有効期間と存続）の指摘**: 秘密保持義務が「永続的」である点に対し、乙の活動を制限するリスクを指摘し、3年に限定する修正案を提示（精度：優秀）。

## 3. クラウド ⇄ オンプレミス移行に伴うトレードオフ分析

| 評価軸 | クラウドAPI (Gemini API) | オンプレミス (Ollama / Local LLM) |
| :--- | :--- | :--- |
| **セキュリティ** | 送信データのオプトアウト設定等が必要（要確認） | **完全クローズド (100%データ機密保持)** |
| **初期投資コスト (CapEx)** | なし（API利用開始のみ） | サーバー/PC機材費（EVO-X2等） |
| **運用コスト (OpEx)** | 従量課金（リクエスト数・トークン数に比例） | 電気代・保守管理工数のみ（**API利用料ゼロ**） |
| **処理速度** | 高速・超低レイテンシ | ハードウェア性能に依存（GPU/VRAMが必須） |
| **管理負荷** | ゼロ（フルマネージド） | インフラの保守、モデルのアップデート管理 |

## 4. アーキテクトとしての提案（TCOとセキュリティの最適化）

エンタープライズのリーガルチェック要件（機密保持）とコストパフォーマンスを両立させるため、以下の**ハイブリッド・アーキテクチャ**を提案します。

```
                    [ 契約書ファイル (機密) ]
                               │
            ┌──────────────────┴──────────────────┐
            ▼                                     ▼
   【ケースA: 極めて高機密】             【ケースB: 一般的な契約書】
   ・社外秘、未公開特許情報等             ・標準的なNDA、取引基本契約書等
            │                                     │
            ▼                                     ▼
      [ オンプレミス ]                       [ クラウド ]
   Ollama / Gemma 2 27B                  Gemini 3.5 Flash
 (完全オフライン / 閉域網)               (API利用 / 超高速処理)
```

1. **本番機（EVO-X2）による「ローカル推論」の常時運用**:
   社外秘かつ顧客の財務・特許情報が記載されているような高機密な契約書のチェックは、インターネットから完全に隔離された EVO-X2 内の Ollama サーバーで実行。
2. **クラウドAPI（Gemini）との併用（ハイブリッド設計）**:
   重要度が低く、処理速度を優先する定型的な契約書（一般的なNDAなど）は、クラウドの Gemini API に逃がすことで、処理キューの滞留を防ぎ、ユーザー体験を損なわない運用設計を行います。
